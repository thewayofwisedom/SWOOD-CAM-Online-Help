NESTING PROJECTS PARAMETERS

The Nesting project parameters are the main parameters of the SWOOD Nesting algorithm.

They are accessible in the parameters section in the Nesting projects window after selecting the highest level of the Nesting project, the " Nesting project".

 

Nesting's project parameters are separated into 5 categories:

 

Nesting parameters

Nesting parameters are the main parameters that will affect the nesting algorithm.

  •  Rectangular nesting: If this option is checked, the algorithm will create a rectangular outline around the non-rectangular nested parts to nest them as rectangles.

  •  Sequence method: The "Surface" method sorts the parts to be nested starting with the parts having a largest surface. The "Perimeter" method sorts the parts starting with the parts having a largest perimeter.

  •  Resolution: Resolution of the shape of the part to be nested. The smaller the resolution, the higher the nesting accuracy, but the longer the computation time.

 

  •  Tool diameter: Defines the standard tool diameter for the whole project, it will be possible to define a specific tool diameter for particular parts.

 

  •  Offset: Also called CL, defines the extra distance between two nested parts.

 

  •  Rotation step: Defines each rotation step for parts to be nested. For rectangular parts, it should be set to 90°, for faster computation. The smaller the rotation step, the better the optimization of the parts for complex geometries, but the longer the computation time.

 

  •  Part distance type: There are two distance types between parts:

"D1+D2+CL" which corresponds to the calculation of: tool diameter of the first part (D1) + tool diameter of the second part (D2) + the offset (CL). In most cases, D1 and D2 are identical.

"MAX (D1, D2) + CL", which corresponds to the calculation of: the largest diameter between D1 and D2 + the offset (CL). This type creates a smaller spacing between parts.

  •  Fit part in part: If this option is checked, the algorithm will allow nesting of small parts within the inner contours of a larger part. This option allows only "D1+D2+CL" as part distance type.

 

  •  Best nesting direction: If this option is checked, the algorithm compares the nesting in both directions (X or Y) and chooses the one that generates the largest rectangular scrap.

 

  •  Fillers nesting settings: For some companies, certain parts, often requested, must be produced in excess. In order to define a part to be nested as a "filler part", go to the Maximum quantity of a part to be nested. The options are :

             - "Entire sheet": -Adds "fillers" to the entire empty space of the board. To be chosen when you want to use the whole board.

             - "Till the nested height": Adds "fillers" to the last part before the main scrap. To be used when you want to add "fillers" while keeping the main scrap to be used later.

             - "Till the nested height for last sheet only": Adds "fillers" to the last part before the main scrap, but only on the last board, because often the last board has a very large main scrap that may be interesting to reuse later. The rest of the boards follow the "Entire sheet" mode.

 

  •  Use fillers on last sheet: If this option is checked, the algorithm fills the last board of the nesting with "fillers" according to the chosen filling mode.

 

  •  Tolerance outlines: Tolerance in spline decomposition. The smaller the tolerance, the higher the accuracy of the spline, but the longer the computation time.

 

  •  Part thickness tolerance: Allows nesting of parts with a difference in thickness compared to the thickness of the board. For example, a part of 18,9mm will be nested on a 19 mm board if the tolerance is at least of 0,1mm.

 

  •  Ignore null thickness top/bottom: If this option is checked, the algorithm only considers the main panel material when the laminate thickness is zero.

 

Numbering parameters

The numbering is a way to identify the parts on the boards, this numbering will be computed automatically during the nesting computation, it can be modified in the Manual nesting and can be exported in the report.

  •  Mode: Defines the nesting numbering mode:

             - "None": No automatic numbering.

             - "Based on order": Same numbering as the machining order. (lien à mettre)

             - "Corner/Direction": The numbering starts from the chosen corner and in the chosen direction.

             - "Corner/Direction based on boards": The numbering starts from the corner and the direction chosen in the board library.

  •  Direction: (Option modifiable only with the Corner/Direction mode) Direction in which the numbering is done.

  •  Corner : (Option modifiable only with the Corner/Direction mode) Corner of the board where the numbering starts.

 

  •  Tolerance : Defines the tolerance in the part detection according to the direction and the chosen corner.

Strategy parameters

Strategy parameters are used to improve the holding of the parts on the nesting boards as much as possible.

It is directly linked to the  Part Type Configuration.

  •  Optimal holding: Activates the part holding function.

 

  •  Part types: (Option modifiable only when the Optimal holding is activated) Possibility to select one of the part types strategies saved in the  Part Type configuration window.

 

Machining parameters

The Machining parameters allow you to apply machining settings even before the machining phase is created.

  •  Quantity: Number of copies of nesting assemblies.

 

  •  Milling order: Defines the milling order of the nesting project, the three options of this parameter are :

             - "None": The milling order is not generated.

             - "Near": The milling order is generated from one part to the closest part to the previous one.

             - "Surface": The milling order is generated from the part with the largest surface to the part with the smallest surface.

This order can be changed in the Manual nesting window.

  •  Default CN: Sets the default machine (post-processor) used in SWOOD CAM.

 

  •  Enable automatic machining insertion: Automatically inserts machining operations (defined in the Automatic machining library) in the milling phase of the nesting result.

 

  •  One program for same sheets: Generates only one Nesting assembly for sheets that have exactly the same part layout. In front of the machine, the operator will have to restart the program as many times as the quantity to produce.

 

  •  Insert not nested parts: Integrates non-nested parts (due to an error), in red, into Nesting assemblies without integrating them on the Nesting board.

 

  •  Lead in/out on the middle of the longest edge: Place the beginning and end of the outline of each part in the middle of its longest edge.

 

 

Tooltip

Tooltips are accessible when hovering over Nesting parameters, they give additional information about these parameters.
